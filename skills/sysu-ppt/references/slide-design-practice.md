# 学术幻灯片设计实践

本参考把演示设计原则转成 `sysu-ppt` 的选页、写作、配图与验收规则。它不提供可机械套用的装饰模板；官方模板负责品牌一致性，本文件负责信息结构、视觉证据和版式节奏。

## 1. 先写逐页合同

生成内容前先列故事板，每张实质内容页填写：

| 字段 | 要求 |
| --- | --- |
| 标题 | 一条结论式名词短语，不能只是“研究背景”“结果”“方法” |
| 主信息 | 听众离开本页时应记住的一件事 |
| 证据或结构 | 支撑主信息的真实数据、文献事实、比较、流程、机制、公式或实例 |
| 解释或边界 | 证据意味着什么，或在什么条件下成立；只保留一条 |
| 主视觉 | 统计图、证据图、结构图、表格、公式、代码、真实截图或无 |
| 版式 | 从内容关系映射到现有 API，不从“看起来不同”出发 |
| 口头内容 | 需要讲但不应写到页面上的定义、理由、细节和过渡 |

一页一条主信息。内容丰富指“主信息 + 最强证据或结构 + 必要解释或边界”闭环，不是元素更多、文字更长或每页配图。

页数先服从用户要求和汇报时长，再由论证链与证据量自然决定。公开指南中的“一分钟一页”和仓库蓝本中的页数都只能用于初步估算，不能驱动补页、删减必要证据或固定每页内容。

## 2. 内容关系到版式的映射

| 内容关系 | 首选构图 | API | 避免 |
| --- | --- | --- | --- |
| 单一结论 + 关键证据 | 结论式标题 + 大幅证据视觉 + 一条解释 | `sysu_add_image_caption()` / `sysu_add_image()` | 把图缩成角落，旁边堆完整段落 |
| 定义、方法解释、证据评述或边界 | 1–3 个完整段落；论点明确，句间关系连续 | `sysu_add_text()` / `sysu_add_two_text()` + `prose()` | 把连续论证拆成标签墙或项目符号墙 |
| 两组真正并列或对照 | 双栏或比较表，维度和顺序一致 | `sysu_add_two_text()` / `sysu_add_table()` | 左右栏比较维度不一致 |
| 3–4 个同层级对象 | 克制卡片，仅保留名称、差异和适用边界 | `sysu_add_cards()` | 把连续流程或长段正文切成卡片墙 |
| 机制、流程、路径或层级 | 一张主图占正文区 55%–90%，必要时复用母图逐步高亮 | `sysu_add_text_image()` / `sysu_add_image*()` | 用项目符号复述箭头关系 |
| 统计结果 | 发表级统计图 + 一条结果解释；数字从结果单源读取 | `publication-figures` + `sysu_add_image*()` | 重画或美化出并不存在的趋势 |
| 方法比较、文献证据或计划 | 对齐的表格、矩阵或时间结构 | `sysu_add_table()` / `sysu_add_text_table()` | 彩色大表头、过密网格或逐格长句 |
| 代码与操作 | 关键代码 + 目的或输出说明 | `sysu_add_code()` | 完整脚本截图或不可读终端图片 |

## 3. 版式节奏与视觉统一

- 任意连续 4 张实质内容页至少使用 2 种主构图；除非是在同一母图或同一分析结果上进行有意的逐步讲解，不得连续 3 张完全同构。
- 全篇通常覆盖至少 4 类主构图，但不为达成数量强行轮换。左右图文换边不算新的信息结构。
- 每页只设一个视觉焦点。大图、表格、公式、代码或核心结论只能有一个主角，次要内容降级为一条解释或题注。
- 统一标题、字体、主色、题注、图号和页码；变化来自信息关系、比例和视觉权重，不来自随机色块、图标、阴影、渐变或字体。
- 单页通常保持 80%–90% 的白色或近白中性区域，品牌绿用于标题、规则线、关键边框和少量强调；这是色彩面积约束，不是要求页面空置。红色等第二强调色通常低于页面面积的 5%。卡片不按项目随机换色，层级依靠字号、字重、位置和克制的明度差。
- 白空间用于分组和强调，不要求每个区域都有内容。主体过小而页面仍显空时，先放大主视觉或拆掉次要区域，不补装饰。
- 只有 2–4 条短文本且占不到正文区一半时，先从材料补充有来源的证据、解释或适用边界；无内容可补就与相邻页合并或改成双栏、比较表、主视觉、关键数字或卡片。仍需独立成页时，增大字号、行距和段间距，并用 `top`/`height` 调整纵向位置，不能接受“上方几行字、下方整片空白”。
- 需要连续解释复杂方法时复用同一母图，只改变当前高亮模块；不要每页换一套画风、节点或方向。

## 4. 文字与信息密度

- 标题使用结论式名词短语，尽量一行完成；避免“背景”“研究结果”“问题”等无信息标题，也避免反问和口语。
- 文字密度按页面功能分层：结论、比较和决策页通常用 2–4 条精炼大字；定义、方法解释、证据评述和边界页允许 1–3 个完整段落。单段围绕一个论点展开，通常控制在约 2–5 个显示行；超过后先检查是否可删、可分栏或可拆页。
- 不把所有页面都做成短标签。存在因果、条件、转折或限定关系时，用 `prose()` 保留完整句法；只有真正并列、可快速扫读的内容才用编号、项目符号或卡片。
- 页面只写讲述时必须看见的文字。逐字朗读的正文、图中已表达的关系和标题的重复解释全部删除。
- 既有模板标题默认 24 pt；正文 16 pt 仅为密集页下限，内容允许时优先 18–20 pt，短结论或关键数字可用 21–24 pt。最终以 1280 × 720 导出图和教室后排可读性验收。
- 正文与背景对比度至少 4.5:1；颜色编码必须同时提供形状、线型、标签或位置线索，不能只靠红绿区分。

## 5. 配图的准确性与审美

- 视觉只接受四种功能：呈现真实对象、组织关系、解释机制或提供证据。纯装饰图、剪贴画、无语义图标和伪造界面不进入实质内容页。
- 先核验对象、数字、标签、箭头、方向、时间和来源，再评价构图、配色与细节。高优先级事实错误未解决时，不得因“更好看”采用该图。
- 外部图必须来源可追溯、分辨率足够且许可允许使用；不得放大超过原始分辨率。真实界面和分析产物使用实际截图。
- 非统计视觉按 `research-visuals` → `imagegen` 生成并验收；统计图走 `publication-figures`；科研原始图像不做生成式重绘。
- 单页通常使用一张高质量主图，不拼贴多张风格不一的小图。多图确有比较意义时，统一裁切比例、视角、尺度、题注与对齐基线。
- 题注写对象和证据身份，解释写正文。图内只保留理解对象、关系、条件和关键状态所需的最小文字。

## 6. 交付前节奏检查

在幻灯片浏览视图和 1280 × 720 导出图中同时检查：

1. 只读标题能否还原完整论证顺序。
2. 每页能否用一句话说出主信息，并指向唯一的支撑证据或结构。
3. 是否出现连续 3 张完全同构、连续卡片墙或连续纯文字页。
4. 主视觉是否足够大，是否存在可删除而不损失含义的装饰。
5. 图、表、公式和数字是否与来源一致，题注和来源是否可追溯。
6. 是否有文字过小、连续长段、过量要点、低对比或只靠颜色传意。
7. 是否出现标题无意义换行、单字或单个标点独占末行、卡片正文只占一角，或某栏明显重于另一栏；命中后优先精炼措辞、调整栏宽或改布局，不缩小字号。

## 7. 在线依据

以下页面于 2026-07-20 核验，仅吸收可迁移原则，不复制其模板或素材：

- [Harvard Catalyst: Slides](https://catalyst.harvard.edu/writing-communication-center/visualize-science/slides/)：assertion-evidence 结构、强标题、视觉证据、少用项目符号、高分辨率图片与一致外观。
- [Harvard T.H. Chan School of Public Health: Slide checklist](https://hsph.harvard.edu/research/health-communication/resources/slide-checklist/)：每页一句式标题、2–4 个项目、连续文字不超过 2 行、充足留白、对比度至少 4.5:1、非纯颜色编码；其中字数建议作为快速扫读基线，不覆盖本文件对解释页完整段落的内容驱动规则。
- [MIT NSE Communication Lab: Slide Design](https://mitcommlab.mit.edu/nse/commkit/slide-design/)：一页一个信息点、标题写明主旨、内容支撑主旨、提高信噪比、图表注释、统一格式和自然转场。
- [UC San Diego: Evidence-Based Presentation Design Recommendations](https://multimedia.ucsd.edu/best-practices/presentation-design.html)：图像优先但禁止装饰图，工作记忆“4 个组块”原则，显著性提示，高分辨率和无障碍替代文本。
