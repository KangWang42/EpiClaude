# 科研证据调研

把问题转化为可复核的检索协议、来源记录、证据矩阵与适用性结论。所有正式引文必须经过来源核验；检索片段和二手转述只用于发现候选文献。

## 1. 锁定决策问题

先写内部问题卡：

- 构念：要测量、解释或预测什么；
- 研究对象与场景：人群、疾病、地区、时间和用途；
- 数据结构：横断面、纵向、重复测量、时间结局、信号或文本；
- 候选方法或指标；
- 决策标准：效度、偏倚、可解释性、可实施性、外部与内部验证要求。

问题涉及分组、终点、纳排或主分析方法且定义不清时，先问用户。文献不能替用户决定研究口径。

## 2. 预注册检索协议

检索前按 [references/search-protocol.md](references/search-protocol.md) 写明：

1. 数据库与权威网站；
2. 检索日期与覆盖时间；
3. 每个数据库的检索式；
4. 语言、研究类型、人群与全文纳排条件；
5. 去重和筛选方法；
6. 停止条件。

“最新”不等于只取最近文章。保留奠基方法论文，同时重点检索近 3–5 年的验证、更新、指南和系统综述。需要实时检索时记录精确日期。

## 3. 按问题调整证据层级

优先级不是固定排名，按决策问题组合：

- 权威指南、标准与共识：定义规范、临床或报告要求；
- 系统综述与 Meta 分析：总体证据与异质性；
- 方法学原始论文：方法定义、假设与估计目标；
- 目标领域验证研究：实际性能、可重复性与适用人群；
- 相邻领域研究：仅作为间接证据，必须做可迁移性判断。

方法的奠基论文不能被较新的二手综述替代；指南也不能替代目标数据中的内部验证。

## 4. 核验每条来源

对拟采用的每条引文逐项核对：

1. 标题、作者、年份、期刊或发布机构；
2. DOI、PMID 或稳定 URL；
3. 文章类型、研究对象、样本和主要方法；
4. 正文实际支持的结论与限制；
5. 撤稿、更正、勘误或版本状态。

优先用出版商页面、PubMed、Crossref、指南发布机构和其他稳定主源交叉核验。无法取得全文或无法核对关键结论时标记“未核验”，不得作为正式依据。发现 DOI 错误、撤稿或来源不明时剔除并记录原因。

## 5. 提取证据矩阵

使用 [references/evidence-matrix.md](references/evidence-matrix.md) 的字段逐条提取，不把多个研究合成一句无来源结论。至少记录：

- 决策问题与候选方案；
- 证据类型、研究对象或领域、直接性和时间；
- 来源标识与核验状态；
- 主要结论、关键假设和限制；
- 对当前项目的映射与论断强度。

来源之间不一致时展示差异，检查人群、定义、测量、模型、时间窗和偏倚来源，不用简单票数表决。

## 6. 判断跨领域可迁移性

借用相邻领域方法前逐项回答：

1. 构念是否相同，而非名称相似；
2. 测量尺度、采样过程与数据生成机制是否兼容；
3. 方法假设在目标数据中是否可检验并满足；
4. 目标样本能否进行内部验证、校准或复现；
5. 间接性要求如何降低论断强度。

任一关键项无法判断时，不建议直接迁入主流程；把它登记为需隔离验证的候选方法。

## 7. 特殊决策门禁

### 结局指标或综合评分

明确区分：

- 理论驱动：构念与权重来自理论或共识；
- 数据驱动：结构或权重由当前数据学习；
- 预测驱动：以预测性能为目标优化。

无论哪类，都说明构念效度、内部验证、稳定性分析、敏感性分析与外部验证需求。数据驱动或预测驱动评分不得包装成既有临床标准。

### 异常值调整

调整观测值必须同时核对原始记录、单位和录入规则、重复测量或相邻时间点及分布证据。文献只能说明合理范围或核查方向，不能单独证明某个观测值应改成多少。

## 8. 输出与衔接

交付：

1. 检索协议与检索日期；
2. 纳排后的来源清单及未核验、撤稿、更正记录；
3. 证据矩阵；
4. 候选方案比较、直接性与可迁移性判断；
5. 推荐决策、证据强度、限制和需内部验证的项目；
6. 可直接交给下游的已核验引文标识。

进入分析时，把方法假设与验证要求交给 `biostat-principles`、`r-biostats` 或项目 Python skill；进入写作时把已核验引文与证据边界交给 `academic-publishing`。本 skill 不代跑模型，也不代写论文正文。
