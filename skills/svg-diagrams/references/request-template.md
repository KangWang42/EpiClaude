# SVG 图解规格模板

在生成 SVG 前，将用户材料整理为以下规格。用户已经提供的内容原样保留；缺失但不影响拓扑的项使用默认值，缺失且会改变关系或结论的项先询问。

## 基本信息

```text
用途：内部 PPT / 论文投稿 / 报告
目标载体：全页 / 半页 / 单栏约 85 mm / 双栏约 170 mm / Word 版心
语言：中文 / 英文
图类：队列筛选 / 概念机制 / 技术路线 / 层级结构 / 包含关系 / 时间轴 / 矩阵 / 系统架构
视觉配置：journal-flow（队列筛选/纳排/CONSORT）/ editorial（其余非统计图解）
色彩模式：semantic-role（默认，两类语义主色）/ semantic-role-critical（存在真实警示、失败、异常、不良结局或关键状态时增加第三类）
版式：A 层级汇聚 / B 谱系时间轴 / C 嵌套包含 / 线性流程 / 树形层级 / 矩阵 / 容器架构
背景：#FFFFFF 或 #FDFDFB
图注：外置 / 嵌入；编号与标题
输出：SVG + 同名 PNG；按需派生 PDF/EMF
```

| node_1 |  |  | biology/exposure/covariate/risk/outcome/nonlinear | primary/secondary/critical/neutral |  |  |

标题与副标题规则：只使用来源材料、研究方案或可核验方法文献中的规范术语。只有真实提供次级信息时填写副标题；不得用泛化句、英文翻译、内部流程词、自造缩略语、临时拼接短语或重复标题填空。

## 视觉约束

```text
必须保留：
禁止出现：英文眉题、无来源副标题、卡通图标、装饰圆点、顶部色条、渐变、厚重阴影、大圆角彩色卡片墙
同类配色：
主色相：默认两类；必要时第三类；黑白灰中性色不计；不得超过三类
特殊期刊要求：
```

## 可复用任务指令

```text
根据以上节点表和关系表生成原生 SVG。采用克制、专业、留白充分的期刊插图风格。
先判断图类及应使用 journal-flow 还是 editorial，严格使用 svg-diagrams/references/design-system.md
对应配置的配色、字号、内容框、正交连线和载体规则，
并使用 layout-recipes.md 中与内容匹配的版式。不要为形成固定两行结构而虚构副标题，
不要添加英文眉题、图标、徽章、装饰圆点、渐变、深色顶条、厚重阴影或大圆角彩色卡片墙。
所有图先采用白底、细边框、低饱和浅填充、常规正文和几何优先的期刊研究流程图语法；
流程图默认使用两类语义主色，只有真实警示、失败、异常、不良结局或必须单独识别的关键状态才增加第三类，主色相总数不超过三种，黑白灰中性色不计。不得为制造丰富感而把相邻节点随机换色，也不得默认套用蓝白配色。
不得根据 biology、exposure、covariate、risk、outcome 元数据自动逐类换色；用 data-tone 表达阅读角色。
同层卡片等宽等高，节点间净距不小于卡片高度的 60%，连线只走横平竖直。
在语义元素上写入 data-role、data-category、data-tone 和 data-layer，输出 SVG 与同名 PNG，
并通过 validate_svg.py --profile <journal-flow|editorial> 的对应配置校验后再交付。
```
