# 个人 R 代码风格（软约束 · 服从工作流）

> 定位：写出读起来像本人手写的 R 代码。**这是风格偏好，不是硬红线**。与可复现性、正确性、项目结构红线（编号、registry、单源、实跑验证）冲突时服从红线。
> 提炼自用户真实脚本。目标：直接加载依赖、顺序式组织、管道连续、函数克制、中间对象少、批处理紧凑、注释适度。

目录：1 脚本开头 · 2 管道主线 · 3 函数边界 · 4 中间对象与命名 · 5 批处理 · 6 验证与输出 · 7 版面 · 8 注释 · 9 生态偏好 · 10 红线张力

## 1 脚本开头

- 每个实际使用的包直接写一行 `library()`，让依赖一眼可见。
- 不用 `suppressPackageStartupMessages({ ... })` 包裹包加载，也不为整段脚本统一静音。
- 只加载本脚本实际使用的包；不要把技术栈清单机械复制到每个脚本。

```r
library(tidyverse)
library(haven)
library(survey)
library(openxlsx)
```

## 2 管道作为主线

清洗、筛选、重编码和整理默认用一条连续 `%>%` 或 `|>` 管道推到目标对象，每步独立成行。不要仅为了“分步骤展示”把同一条变换链拆成多个对象。

```r
# 好：从原始数据连续得到分析数据
data_neat <- data |>
  mutate(病程 = floor(as.numeric(difftime(测评时间, 首次发病时间, units = "days")))) |>
  group_by(病历号) |>
  filter(first(病程) <= 180) |>
  ungroup() |>
  rename(年龄 = `年龄（岁）`) |>
  mutate(across(starts_with("是否"), ~ factor(.x, 0:1, c("否", "是"))))

# 差：同一条链被说明性中间对象切碎
data_joined <- left_join(data, ref, by = "病历号")
data_grouped <- filter(data_joined, !is.na(分组))
data_complete <- filter(data_grouped, if_all(all_of(vars), ~ !is.na(.x)))
```

只在以下情况断开管道：该对象是用户要求的明确产物；下游存在真实分支并会复用它；或模型、图、表等对象类型发生实质转换。不要先制造不必要的导出或检查，再以“需要复用”为由保留中间对象。

## 3 函数边界

R 分析脚本默认按数据导入、清洗、分析、导出等顺序分节，不把每一段一次性代码命名成函数。不要沿用 Python 式“所有步骤先定义函数、最后统一调用”的组织方式。

仅在以下情况抽取具名函数：

- 同一逻辑会被调用多次；
- 需要由 `map*()` / `walk*()` 对不同参数重复执行；
- 是跨脚本复用的稳定工具，如结果渲染或统一出图；
- 算法本身复杂，独立测试边界能明显降低错误风险。

只调用一次的格式化、清洗或导出步骤优先直接写入管道。匿名函数 `\(x) ...` 是批处理语法，不等于过度函数化。像“定义一次绘图函数，再用 `map2()` / `walk2()` 批量生成”属于合理复用。

## 4 中间对象与命名

- 能进入管道就不另起对象。确需落地时，用短而明确的语义名，不用 `df1`、`tmp`、`mid2`、`result3`。
- 常用数据里程碑为 `data`、`data_neat`、`data_completed` / `data_neat_imputed`、`data_baseline`、`data_repeat`；名称应让人一眼看出用途。
- 不为每个筛选、连接、格式化和核验步骤各留一个对象。模型、最终表、最终图或被真实分支复用的数据可以保留。

```r
data_baseline <- data_neat_imputed |>
  group_by(身份证号) |>
  slice_min(病程, n = 1, with_ties = FALSE) |>
  ungroup()
```

## 5 批处理：先定义控制向量，再直接迭代

- 单输入并返回对象用 `map()`，成对输入用 `map2()`，需要名称或位置用 `imap()`。
- 保存文件、写表等只产生副作用的任务用 `walk()`、`walk2()` 或 `iwalk()`，不要先生成无用返回列表。
- `map*()` / `walk*()` 只表达迭代语义，默认仍为串行。遇到计算密集且彼此独立的批次时，先按 `../../biostat-principles/references/compute-resources.md` 完成并行门禁，再使用项目已有后端对应的并行映射；轻量批处理不为提高 CPU 占用机械改成并行。
- 批处理前集中定义 `vars`、标签向量、参数向量或列表；迭代体直接完成目标，不堆中间过程。
- `for` 并非禁用。顺序执行更清楚时，使用 `seq_along()` 遍历预先定义的向量或列表；避免 `1:n`、循环内连串 `if` 和无意义累积对象。
- 批量列变换用 `across()`；多分支重编码用 `case_when()`；配置差异优先放入命名向量或列表，少写重复 `if/else`。

```r
vars_to_plot <- c(
  "ICF_RS总分" = "ICF-RS 总分",
  "ICF_身体功能" = "ICF-身体功能",
  "ICF_日常活动" = "ICF-日常活动"
)

walk2(names(vars_to_plot), vars_to_plot, \(var_name, var_label) {
  ggsave(
    filename = str_glue("31_plot_line/plot_line_{var_name}.png"),
    plot = plot_line(data_neat_imputed, var_name, var_label, cus_span = 0.1),
    dpi = 300
  )
})
```

## 6 验证与最终输出分离

- 必须在 RUN / VERIFY 阶段实跑脚本、核对样本量与关键范围并全量扫描异常，但不默认把核验过程展示成分析正文。
- 不为“证明检查过”在交付脚本中堆硬编码样本量的 `stopifnot(nrow(...) == 678)`。这类当前批次核验放在运行检查中；只有研究设计要求长期保护的稳定不变量才保留在脚本里，并优先写成关系约束而非冻结行数。
- 最终脚本不留调试 `cat()` / `print()`、逐步对象展示或被注释掉的检查代码。长批处理确需进度提示时只保留简短且有用的信息。
- 不用全局 `suppressWarnings()` 掩盖异常。只有已定位、确认不影响结果且范围明确的已知警告，才在具体调用处窄范围静音；警告原因仍在验证记录中闭环。

## 7 版面与缩进

- 每个管道步骤独立成行；长向量和多参数调用按语义换行并保持层级清楚。
- 遵循 tidyverse 风格：2 空格缩进，`<-` 赋值，运算符两侧留空格。
- 同一脚本统一使用 `%>%` 或 `|>`；维护旧脚本时跟随原文件，不为统一符号扩大修改范围。

```r
费用指标 <- c(
  "西药", "诊查费", "护理费", "化验费",
  "中成药", "理疗", "总金额", "自费"
)
```

## 8 注释规范

- 分节使用 RStudio 折叠标记，如 `# 数据导入 --------------`、`# 可视化 --------------`。
- 注释简短、中文，解释为什么采用该口径或步骤；不逐行翻译语法，不写“定义变量”“调用函数”“保存结果”这类代码已说明的废话。
- 不留 `# print(x)`、大段注释掉的旧代码或生成过程说明。正式项目旧实现按归档规则进入 `09_backup/`。

```r
# 数据清洗 --------------
data_neat <- data |>
  filter(first(病程) <= 180) |>  # 仅纳入 180 天内首评
  mutate(年龄段 = if_else(年龄 < 60, "<60", ">=60"))
```

## 9 生态偏好（可选）

本人惯用 `fs`（`path()`、`dir_ls()`、`path_file()`）、`str_glue()`、有序 `factor()` 和 `compareGroups::descrTable()`。包选择仍以可复现和团队可读为先；`bruceR::set_wd()` / `import()` 等便利函数若引入重依赖或隐式路径，正式项目优先使用项目既有的相对路径方案和明确的读写函数。

## 10 与硬红线的张力

| 个人习惯 | 冲突点 | 处理 |
|---|---|---|
| `rm(list = ls())` 开头 | 脚本化复现不应依赖清空交互环境 | 可跟随既有脚本，但不作为复现前提 |
| `descrTable() |> export2word()` | 正式项目可能锁定 xlsx、registry 与编号 | 服从项目输出契约 |
| 最终脚本简洁无核验展示 | 必须实跑并扫描 error / warning | 核验在 RUN / VERIFY 闭环；仅保留必要的长期不变量 |
| 线性长脚本 | 可维护性 | 按分析逻辑分节；不以行数为由自动函数化或拆成多文件 |

风格服务可读性，不与原始数据只读、口径确认、结果单源、相对路径、实跑验证和异常闭环对抗。
