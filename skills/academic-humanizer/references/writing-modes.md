# 基金、审稿回复与科研往来模式

## 目录

1. 基金申请
2. 审稿回复与 rebuttal
3. Cover letter
4. 编辑部与科研邮件
5. 完成检查

## 1 基金申请

基金文本的核心是愿景与可行性同时成立。不要按论文的克制尺度删除所有目标性语言。

- 首页优先回答：问题造成什么具体后果、当前缺口是什么、核心思路是什么、分目标是什么、完成后改变什么。
- 每个 aim 或研究内容以问题或结果为中心，不以“应用某算法”作为目标本身。
- 分目标尽量相互独立；存在依赖时明确失败后的替代路径。
- 每项承诺旁边放可行性依据：先导数据、已发表工作、理论基础、合作条件、数据来源或阶段性验收标准。
- 先导结果、项目经历、合作单位和支持信只能使用用户提供的真实信息。
- 现行页数、格式、截止日期和评审条款必须查资助机构官方文件，不凭记忆写。

## 2 审稿回复与 rebuttal

建立内部响应矩阵：`评论编号 | 审稿人要求 | 采取的动作 | 证据或理由 | 稿件位置 | 核验状态`。

逐条执行：

1. 原样保留评论并编号，不合并、不漏答。
2. 先用一句话给直接答案，再说明采取的动作或不同意的依据。
3. 已修改时给出准确位置和必要的修订文字；未修改时不得声称“已修改”。
4. 新增分析时，先核对代码、结果单源、表图和正文均已同步，再写入回复。
5. 不同意时承认问题的合理部分，给方法学、数据或范围依据，并说明稿件中增加了何种澄清。
6. 无法完成的要求应说明真实限制和对结论的影响，不编造分析、数据或附件。

语气保持克制、专业、直接。每条回复不重复整段致谢，不使用防御性、讽刺或讨好语气。

## 3 Cover letter

按以下顺序写：稿件题名与类型、具体研究缺口、最重要的真实发现、与期刊范围的匹配、原创性与投稿声明、
通讯作者信息。不要复述摘要，不写无证据的“首次”“突破”或笼统的“读者会感兴趣”。编辑姓名必须来自官方页面。

## 4 编辑部与科研邮件

适用于投稿询问、延期申请、撤稿说明、数据或材料请求、合作沟通和共同作者确认。

- 主题行写明稿件编号、题名缩写和所求事项。
- 首段直接说明来信目的；中段给必要事实、日期和附件；末段写清希望对方采取的动作。
- 延期申请给出具体新日期和简短真实原因，不写长篇辩解。
- 错误更正或撤稿说明区分事实、影响范围和已采取措施，不淡化也不扩大。
- 数据请求明确用途、范围、伦理或署名安排；不得暗示对方已同意。
- 共同作者邮件区分“征求意见”“请求批准”和“告知结果”，避免含糊授权。

## 5 完成检查

- [ ] 基金承诺均有可行性依据或替代路径。
- [ ] 审稿意见逐条响应，声称的每项修改均能在稿件中定位。
- [ ] 新增分析已同步结果单源、表图、正文和回复。
- [ ] 不同意意见时给出证据，不以语气代替论证。
- [ ] 投稿信和邮件中的期刊名、编辑名、稿件号、日期、附件与声明均真实可核。
- [ ] 数字、引用、公式、作者信息和结论方向未被编辑过程改变。
