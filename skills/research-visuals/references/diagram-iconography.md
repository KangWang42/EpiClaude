# 科研流程图图标、配色与内容密度

本规范用于流程图、研究设计、技术路线、方法架构和图形摘要中的 icon/pictogram。默认从“无图标也能读懂”开始，只在图标能更快识别真实对象、操作或产出时加入。图标辅助标签，不替代文字、箭头和内容合同。

## 1. 调研依据

| 来源 | 可迁移原则 |
| --- | --- |
| [PLOS Computational Biology: Ten Simple Rules for Better Figures](https://journals.plos.org/ploscompbiol/article?id=10.1371/journal.pcbi.1003833) | 图形服从受众和核心消息；投影时元素要更粗、更大、对比更强；颜色必须有明确用途 |
| [PLOS Biology: Creating clear and informative image-based figures](https://journals.plos.org/plosbiology/article?id=10.1371/journal.pbio.3001161) | 检查灰度与色觉缺陷可读性；颜色与形状、线型或标签双重编码 |
| [W3C Images Tutorial](https://www.w3.org/WAI/tutorials/images/) | 区分信息型与装饰型图像；装饰元素过多会增加噪声，复杂图需要等价文字说明 |
| [W3C G207: Contrast for icons](https://www.w3.org/WAI/WCAG22/Techniques/general/G207) | 理解内容所必需的图标与相邻背景至少保持 3:1 对比度 |
| [Microsoft Fluent 2 Iconography](https://fluent2.microsoft.design/iconography) | 图标使用可识别的字面隐喻；小尺寸减少细节；修饰符不能引入视觉或语义复杂度；着色通常只用一种颜色 |
| [IBM Pictogram Usage](https://www.ibm.com/design/language/iconography/pictograms/usage/) 与 [Design](https://www.ibm.com/design/language/iconography/pictograms/design/) | 多数场景使用简单、统一的 productive pictogram；复杂 expressive pictogram 只在少数焦点位置使用；保持网格、留白、笔画和色彩一致 |
| [GOV.UK Design System: Images](https://design-system.service.gov.uk/styles/images/) | 多数情况下避免无必要图标；一个图标不要同时代表多个概念 |

以下数量与密度是 EpiAgentKit 对科研技术路线的本地操作性启发式，不是期刊或无障碍标准。

## 2. 是否使用图标

每个候选图标必须同时通过四个问题：

1. **语义贡献**：移除图标后，读者识别真实对象、操作或产出的速度是否明显下降。
2. **主题对应**：图标是否对应项目实际出现的人群、样本、设备、信号、数据库、分析、验证或输出。
3. **最终尺寸**：在投影、印刷或页面 100% 缩放下是否仍能一眼辨认；需要放大才能看懂就删除。
4. **系统一致**：是否能与其它图标保持同一笔画、视角、圆角、填充、容器和颜色逻辑。

任一问题为否，使用文字、编号、形状、真实微型图元或分组边界替代。抽象的“创新、价值、科学性、智能、先进”不使用灯泡、奖杯、火箭、脑、芯片或发光 DNA 代替。

## 3. 数量与分布

| 路线复杂度 | 默认策略 | 本地上限 |
| --- | --- | --- |
| 3–4 个高层阶段，且每阶段都有具体对象 | 可每个主阶段一个简单图标 | 全图最多 4 个 |
| 5–7 个主阶段的常规技术路线 | 只标输入、关键处理、验证、输出等锚点 | 通常 2–4 个，不逐节点配图 |
| 多分支、多层或文字较密的路线 | 仅为大区或关键转折提供锚点 | 全图 0–3 个 |
| 论文单栏、窄版标书或最终尺寸较小 | 优先不用图标 | 0–2 个，无法清楚辨认则为 0 |

- 同一语义层级要么都使用图标，要么只给有明确证据优先级的锚点使用，不能随机装饰部分节点。
- 图标不重复节点标签已经清楚表达的抽象概念，不为填空加入图标。
- 图标总视觉权重低于阶段标题和主路径；不能抢过箭头、关键标签和结果输出。

## 4. 复杂度等级

| 等级 | 形式 | 适用位置 |
| --- | --- | --- |
| L0 | 无图标，仅文字、编号、边界和箭头 | 精确流程、密集路线、论文小尺寸图 |
| L1 | 单一字面对象、单色、轮廓清楚、无内部纹理 | 普通节点和 PPT 技术路线锚点 |
| L2 | 由 2–3 个必要部件构成的简单 pictogram | 面积较大的主阶段或关键模块 |
| L3 | 有层次、材质或轻微叙事的表达型图形 | 全图唯一焦点或章节主视觉，不重复放入普通节点 |

Pictogram 在最终导出中低于约 48 px 或等效视觉尺寸时，降为 L1 或 L0。不要把 L2/L3 缩小后塞进节点；不要混用线性图标、彩色扁平插画、照片和 3D 图标。

## 5. 主题映射

优先选择字面、可核验的对象：

| 内容 | 可用图标或微型图元 | 避免 |
| --- | --- | --- |
| 研究人群/队列 | 简化人群轮廓、队列列表 | 无关医生、白大褂、握手 |
| 样本与采集 | 与实际一致的采样管、量表、设备轮廓 | 项目未使用的显微镜或试管 |
| 数据源与链接 | 数据表、数据库、记录连接 | 发光云、悬浮界面 |
| 信号与图像输入 | 真实类型的波形、网格、图像轮廓 | 随机心电、伪热图、假坐标 |
| 清洗与质量控制 | 检查表、缺失标记、对齐或筛选的简单图元 | 抽象盾牌替代全部 QC 内容 |
| 模型与分析 | 与真实方法相符的矩阵、树、网络或回归关系 | 通用 AI 脑、芯片、机器人 |
| 验证与稳健性 | 数据划分、外部队列、重复检验或敏感性分支 | 单独勾号替代验证设计 |
| 输出与解释 | 效应估计、风险分层、报告或决策路径 | 奖杯、火箭、庆祝符号 |

微型图元能更准确表达输入、变换或输出时，优先微型图元，不再叠加通用 icon。

## 6. 风格与配色

- 一张图只使用一个图标家族：统一 outline/filled 选择、笔画粗细、端点、圆角、视角、光照和容器形状。
- 常规技术路线优先 L1/L2 单色或同一色族图标。图标可继承所属阶段颜色；流程图默认使用两类语义主色，只有真实警示、失败、异常、不良结局或关键状态才增加第三类，主色相总数不超过三种，黑、白、灰等中性色不计入。
- 颜色只承担阶段、状态或重点等明确功能。图标不能另建一套彩虹色板，也不能仅靠颜色区分语义。
- 必要图标与相邻背景至少 3:1；图内普通文字按载体尽量达到 4.5:1。避免极细笔画，因为抗锯齿会降低实际对比。
- 用标签、编号、形状、边界或线型与颜色双重编码；签发前检查灰度和常见色觉缺陷模拟。
- 红色只用于真实警示、失败、异常、不良结局或当前强调；普通排除步骤、常规分组和一般终点不自动标红。不使用渐变、霓虹、金属和发光效果装饰普通流程节点。

## 7. 提示词字段

生成前在结构化提示词中加入：

```text
Icon strategy: <none / per-major-stage / semantic anchors only>
Icon budget: <0–4，说明为什么需要这些图标>
Icon map: <stage ID | literal metaphor or micro-visual | semantic contribution>
Icon complexity: <L0 / L1 / L2 / one L3 focal only>
Icon system: <outline/filled, stroke, corner, viewpoint, container, single-color logic>
Icon constraints: no decorative science symbols; no mixed families; no icon without a label; no icon smaller than final recognizable size
```

## 8. 签发检查

- [ ] 每个图标对应真实对象、操作或产出，删除后确会降低识别效率
- [ ] 数量符合路线复杂度，未给每个节点机械配图
- [ ] 图标复杂度适合最终尺寸，L2/L3 未被缩成噪声
- [ ] 图标家族、笔画、视角、容器和颜色统一
- [ ] 图标与文字、箭头、微型图元无重复表达或视觉竞争
- [ ] 关键图标对比充分，颜色不是唯一语义线索
- [ ] 全图主色相不超过三种，默认两类语义色，警示色与实际状态一致
- [ ] 无无关显微镜、DNA、灯泡、芯片、奖杯、火箭或通用 AI 符号
