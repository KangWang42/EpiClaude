# 科研图解内容与视觉规范

本规范从真实流行病学、生物统计、环境健康和临床研究材料中提炼跨主题规律。目标不是复制某套 PPT 风格，而是判断一张流程图、框架图、机制图、研究设计图或技术路线图是否科学、清楚、可核验。

## 1. 好的科研图回答一个问题

一张图先锁定一个主要任务：

- **研究对象是什么**：对象、人群、系统、暴露、疾病或数据源。
- **证据说明什么**：负担、分布、差异、趋势或已有研究结论。
- **变量如何关联**：框架、假设、混杂、中介、效应修饰或反馈。
- **机制如何发生**：触发因素、中间过程、靶点、结局和替代路径。
- **研究如何实施**：样本、采集、纳排、质量控制、变量、模型和验证。
- **方法如何运作**：输入、变换、并行分支、合并、估计对象和输出。
- **项目如何推进**：阶段、依赖、里程碑和交付物。

如果删除图片不影响理解，或图片只能表达“这是科研”“这是创新”，就不应生成。科研图的价值来自结构化信息，不来自装饰性科研符号。

完整论文、项目材料或多图任务先按 `figure-planning.md` 建立来源到图件矩阵。每个采用候选必须有来源锚点、核心命题和独立贡献；章节数量、方法数量和结果数量都不是配图数量。

## 2. 先判定证据属性

| 图类 | 内容来源 | 默认路径 |
| --- | --- | --- |
| 统计数据图 | 真实数据、效应量、时间或空间映射 | `publication-figures` |
| 原始证据图像 | 显微、病理、影像、临床照片、仪器输出、凝胶或原始截图 | 保留原始证据，仅做允许的确定性处理 |
| 证据综合图 | 可核验文献图、统计图、地图、表格或文献矩阵 | 使用真实来源并标注，不生成替代 |
| 解释插图 | 器官、场景、系统、概念或教学示意 | `research-visuals` → imagegen |
| 精确结构图 | 流程、框架、机制、路线、研究设计、图形摘要 | `research-visuals` → imagegen，精度不合格时 SVG 回退 |

统计结果、原始科研图像和解释示意不得混为同一种证据。机制假设、拟议路径和教学插图应明确标为“示意”“研究假设”或“据文献整理”，不能包装为本研究结果。

## 3. 图类与内容契约

### 3.1 概念框架

按研究问题选取：

- 研究对象或人群。
- 核心暴露、干预或预测因素。
- 主要结局。
- 必要的中介、效应修饰和混杂因素。
- 时间顺序、层级或测量层次。
- 关系属性：关联假设、分析调整、拟议机制或已知证据。

观察性研究不把关联箭头画成已证实因果。混杂因素应指向相关暴露与结局，中介位于主路径中，效应修饰用分层或交互位置表达，不把所有协变量平均铺成卡片墙。

### 3.2 机制示意

至少说明：

1. 触发因素或上游暴露。
2. 关键生物、行为、环境或系统过程。
3. 靶器官、功能或中间表型。
4. 临床、公共卫生或研究结局。
5. 证据状态：已知、推测、待验证或本研究关注。

多条路径使用并行轨道，替代机制或不确定路径用不同线型或明确标签。不得加入无依据的分子、受体、器官损伤或因果环节。

### 3.3 研究设计与数据流

按实际研究保留：

- 研究来源、场所、队列或数据库。
- 纳入、排除、随访和分析集。
- 时间点、检测、设备、空间匹配或重复测量。
- 数据链接、去标识、清洗、对齐和质量控制。
- 暴露、结局、协变量、中介和衍生特征。
- 主要分析集和最终研究输出。

纵向设计突出时间轴和重复测量；多源数据先汇入唯一整合节点；样本筛选图逐项核对人数、排除原因和分析集。

### 3.4 方法流程

方法图按“输入 → 操作 → 中间对象 → 分支/合并 → 估计或预测 → 验证 → 输出”组织。并行算法或模型必须说明各自输入和为何合并；图旁文字解释“为什么做”和“解决什么问题”，图内只保留对象、动作、方法和产出。

连续讲解复杂方法时复用同一母图，每张派生图高亮当前模块并淡化其它模块。不得每页更换结构，也不得完全不变地重复整图。

### 3.5 技术路线

技术路线不是模型名清单。按项目实际从下列模块形成闭环：

1. **研究对象与数据源**：人群、队列、数据库、暴露、结局或外部数据。
2. **采集与时间结构**：检测、设备、随访、空间匹配或重复测量。
3. **纳排、清洗与质量控制**：纳排、缺失、异常、数据对齐和质量控制。
4. **变量或特征构建**：暴露、结局、中介、协变量、衍生指标或信号特征。
5. **研究问题与对应方法**：每条分支用“研究问题或估计对象 → 方法”成对表达。
6. **模型验证与稳健性**：交叉验证、外部验证、分层、敏感性、替代定义或消融分析。
7. **结果输出与解释**：效应估计、风险预测、机制路径、模型解释或可交付成果。

推荐主链：

`研究对象/数据源 → 采集与整合 → 纳排/清洗/QC → 变量或特征构建 → 主要研究问题及方法 → 验证/分层/敏感性 → 输出与解释`

多源数据先汇入整合节点，平行研究问题从共同分析集分叉，最终汇入明确的验证或输出节点。每条分支都必须能指出输入、动作和产出。

技术路线中的图标是可选语义锚点，不是完成度指标。按 `diagram-iconography.md` 决定：3–4 个具体高层阶段可每阶段一个简单图标；5–7 个阶段通常只保留 2–4 个输入、关键处理、验证或输出锚点；多分支或密集路线用 0–3 个。图标必须符合课题真实对象与方法，并与文字、箭头和微型图元互补。

### 3.6 图形摘要

只表达一条主命题：`对象或人群 → 方法或机制 → 关键发现 → 含义`。统计数字必须来自单源结果；长方法链、多个次要发现和完整讨论不进入一张图形摘要。

### 3.7 项目计划

使用时间轴或甘特图，节点同时给出时间区间、任务和可核验里程碑或交付物。不用情境插画替代进度关系。

### 3.8 总览加局部与多面板

复杂模型、关键模块或跨尺度机制可采用“母图定位 + 单模块放大”：母图只说明该模块在全流程的位置，局部图展开输入、真实操作、关键变换和输出，并用明确调用线连接。不要同时放大多个模块，也不要让局部图重复母图全部内容。

多面板图先为每个面板写独立问题和贡献，再做两两去冗余：相同数据、相同问题、可直接推导或删除后不影响核心命题的面板应合并或删除。证据优先级不同时可扩大核心面板；等权比较和对称实验保持等权，不强制设置主面板。

## 4. 视觉语法

- 总阶段通常控制在 4–7 个；同阶段可分支，但主阅读方向只选自左向右或自上而下之一。
- 使用分组边界区分输入、处理、分析、验证和输出。流程图默认使用两类语义主色，确有警示、失败、异常、不良结局或关键状态时增加第三类；主色相总数不超过三种，并同时使用位置、编号、形状和明度编码。
- 普通节点控制为 4–14 个汉字、最多两行；数据源节点可放宽至约 20 字。段落、定义和长解释留在图外。
- 红色只用于真实警示、失败、异常、不良结局或当前讲解位置；普通排除步骤、常规分组和一般结局不自动标红。不使用大面积荧光黄、粗红圈或批注式箭头建立层级。
- 箭头只表达数据流、时间先后、处理顺序或明确的研究假设；菱形只用于真实决策条件。
- 平行分支从共同节点整齐分叉，并汇入明确产出。禁止交叉线、反向回指、悬空箭头和不清楚的多对多连线。
- 颜色不是唯一信息线索。阶段和类别同时用标签、边界、位置或形状区分。
- 同类节点等宽、同层共线、间距一致；重点靠位置、字重、边界和有限强调色。图标按 `diagram-iconography.md` 作为少量语义锚点，不使用无贡献的装饰图标。
- 模块框可包含与真实输入、变换或输出对应的波形、频谱、矩阵、网格、特征图或关系网络等微型图元；不能用伪统计图和随机占位图填空，最终尺寸不可辨识时删除。

## 5. 版面与实际尺寸

先确定最终容器，再生成对应比例：

- **论文**：按单栏、双栏或整页物理尺寸检查；小字和密集节点可能需要简化，不靠事后缩放。
- **PPT**：单一概念图通常占侧栏 30%–45%，机制图占 45%–60%，分步方法采用文字 30%–35% 加图 55%–65%，总体技术路线占标题以下 70%–90%。
- **报告或标书**：按版心宽度和 100% 页面缩放检查；一张图只服务一个评审或决策问题。
- **网页**：按真实容器和响应式裁切安全区生成，关键说明不能只放在图内。

多图按共同顶线、底线和尺寸网格排列，逐图题注；不做无基线的悬浮拼贴。图片与正文并排时，主体朝向页面内部，并为文字保留低细节负空间。

## 6. 来源与证据边界

- 外部图、文献截图、统计图和原始证据图像在邻近位置写明图意、证据属性和来源。
- 统计图保留坐标、单位、样本或时间范围、图例和来源，不作为装饰。
- 外部机制图按最终尺寸裁切和简化无关周边信息，但不得改变原意。
- 禁止图库水印、低清截图、缺图占位符、伪论文页面、不可追溯素材和不同画风的随意拼贴。
- 图号、长图题和来源通常由载体原生排版承担；精确结构图只烧录理解关系所必需的短标签。

## 7. 常见失败

- 用灯泡、书本、显微镜、DNA、白大褂或实验室场景表达抽象的“创新”“研究”或“可行”。
- 一页同时混用照片、扁平卡通、统计图、论文截图和不同风格图标。
- 给每个节点机械添加图标，或混用线性、填充、3D、照片和不同笔画家族。
- 把外部机制图、文献表或技术路线缩到最终尺寸不可读。
- 用方法名称填满节点，却没有研究问题、输入、处理、验证和输出。
- 使用大红圈、粗箭头、荧光笔、渐变框、云形批注或残留编辑控制点。
- 流程分支有起点无产出，箭头穿字、交叉或方向含糊。
- 生成图出现伪文字、重复节点、错误器官、无依据机制、水印或“无法显示该图片”。

## 8. 签发检查

- [ ] 图回答一个清楚的科学问题，不能删除而不影响理解
- [ ] 完整论文、项目或多图任务的候选均有来源锚点、核心命题、独立贡献和采用理由
- [ ] 已判定统计、原始证据、证据综合、解释插图或精确结构图属性
- [ ] 内容契约包含理解该图所需的对象、关系、阶段、验证和输出
- [ ] 概念框架未把观察性关联夸大为已证实因果
- [ ] 机制图区分已知、推测与待验证环节，无虚构生物学细节
- [ ] 技术路线从研究对象或数据源闭合到验证与输出，问题与方法成对
- [ ] 技术路线图标数量与复杂度符合最终尺寸，每个图标都对应真实主题且与统一配色系统协调
- [ ] 总览加局部只展开必要模块；多面板已完成两两去冗余且视觉权重符合证据优先级
- [ ] 主阅读方向唯一，节点简短，分支、箭头和阶段边界含义明确
- [ ] 主色相不超过三种，默认至少有两类语义色避免单调；警示色只对应真实警示、失败、异常或不良结局
- [ ] 图内方法名与标签来自来源、方案或规范术语，无内部流程词、生硬直译或自造短语
- [ ] 比例和密度按最终载体生成，缩略图或最终印刷尺寸下仍可读
- [ ] 外部图、统计图和原始证据的图意、属性与来源可追溯
- [ ] 无水印、缺图、低清截图、伪文字、混乱拼贴和无语义装饰
