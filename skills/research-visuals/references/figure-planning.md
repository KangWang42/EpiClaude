# 来源驱动的科研图规划与审校

本规范用于完整论文、项目材料、多图系列、复杂多面板图和带参考图的任务。目标是先决定“是否需要图、图要证明什么、内容从哪里来”，再进入 imagegen。单张图需求已经给出明确内容、比例和用途时，可只执行缩略版图前合同。

## 1. 来源到图件矩阵

完整论文或项目不按章节机械配图。先读取与任务有关的章节、方法、结果、表图、模板和项目规则，建立候选矩阵：

| 字段 | 要求 |
| --- | --- |
| 候选 ID | 稳定短标识，不预先占用正式图号 |
| 科学问题 | 该图只回答一个什么问题 |
| 核心命题 | 读者数秒内应理解的一句话 |
| 来源锚点 | 文件、章节、小节、表、结果键或已核验文献 |
| 证据属性 | 统计数据、原始证据、证据综合、解释插图或精确结构图 |
| 信息角色 | 正文内容图、真实截图、氛围插图、统计图、原始证据或无需图片 |
| 独立贡献 | 相比现有正文和其它图新增什么理解 |
| 图类与载体 | 图类、PPT/论文/标书/报告/网页及实际占位 |
| 实现路由 | `publication-figures`、保留原始证据、`research-visuals` 或 SVG 回退 |
| 文字与模板 | 图内必须逐字呈现的标签；模板必须保留与允许调整的部分 |
| 采用理由 | 为什么图比文字、表格或现有图更清楚 |
| 拒绝理由 | 无独立贡献、证据不足、最终尺寸不可读或其它原因 |

只有同时满足以下条件的候选才进入生成：

1. 有明确来源锚点，或明确标为拟议机制、研究假设、解释示意。
2. 有不可由已有图表或短段落等价替代的独立信息贡献。
3. 在最终载体尺寸下能保持核心关系和必要标签可读。

章节存在不等于需要配图。完整论文也不得默认“每个主要章节一张图”。

## 2. 图前合同

新图、重大重绘和多面板图在生成前锁定八项：

1. **核心命题**：一句话说明图要传达的结论、关系或路径，不写“本图展示了”。
2. **来源与边界**：列出来源锚点、证据属性、可以抽象的内容和禁止推断的内容。
3. **内容链**：每个区域或面板写明其独立贡献；移除后不影响理解的区域应删除。
4. **结构原型**：选定主阅读方向、分支、合并、反馈、时间或层级关系。
5. **载体与几何合同**：确定实际比例、占位、最终物理或像素尺寸、背景和安全区；按 `carrier-specs.md` 核对最长标签的自然字宽、节点内边距、箭头、间距和侧栏是否能放入可用宽度，并锁定禁止窄体压缩和非等比缩放。
6. **风险清单**：预先标出精确文字、数字、因果、解剖、来源、可访问性和裁切风险。
7. **信息与文字角色**：判定正文内容图、真实截图或氛围插图。内容图先对候选文字做删除测试，只将删除后会造成对象、关系、条件、状态或核心结论歧义的文字列入 `Text (verbatim)`；截图锁定真实产物；氛围图才默认无字。定义、理由、完整方法、限制与来源优先移到正文、图注或复杂图长描述。
8. **模板适配**：列出尺寸、品牌、字体、色彩、安全区和必要组件等必须保留项，以及区域数量、阅读方向、图文比例、密度和视觉权重等允许调整项。

轻量改色或明确的单点修正只需锁定“不变量、目标问题、最终尺寸”三项，不重复完整规划。

## 3. 结构原型

原型由内容拓扑决定，不用固定期刊风格：

| 原型 | 适用关系 | 关键要求 |
| --- | --- | --- |
| 单向主链 | 流程、技术路线、研究设计 | 4 至 7 个主阶段，起点与输出闭合 |
| 并行汇聚 | 多源数据、多分支模型、多研究问题 | 共用输入清楚分叉，汇入唯一整合或输出节点 |
| 总览加局部 | 复杂模型、关键模块、跨尺度机制 | 母图交代位置，局部图只展开一个模块，调用关系明确 |
| 证据复合 | 机制或概念图与真实统计/证据面板组合 | 各面板保持证据属性，统一配色与边距，不让示意替代数据 |
| 对照矩阵 | 方法、路径或状态的并列比较 | 比较维度一致，不生成虚构指标或占位数据 |
| 时间或层级 | 随访、项目计划、系统层级 | 时间、依赖或包含关系不能混用普通流程箭头 |

多面板图不强制设置“英雄面板”。证据优先级确有差异时，让核心面板获得更大面积；等权比较、诊断矩阵和对称实验必须保持等权。面板顺序按研究叙事排列，并执行两两去冗余检查：是否使用相同数据、回答相同问题、可由另一面板直接推导、删除后是否影响核心命题。

## 4. 模块内微型图元

复杂流程或模型图中的模块不应只是空框和方法名。允许在模块内放置一个与真实操作对应的微型图元，例如波形、频谱、矩阵、特征图、数据表轮廓、空间网格或小型关系网络，但必须满足：

- 图元直接说明该模块的输入、变换或输出，而非填满空白。
- 统计图缩略图只能来自真实图件或抽象轮廓，不生成似真数值、坐标和趋势。
- 同类模块共享图元语法；关键模块可增加细节，辅助模块保持简洁。
- 最终尺寸下无法辨识时删除，不把复杂细节缩成噪声。

## 5. 参考图解构与编辑目标

先区分图片的角色：仅用于借鉴风格、布局或信息组织的是**非编辑参考**；用户要求修改、重绘或继续修正的图片是**编辑目标**。两者不能混为一类。

### 5.1 非编辑参考解构

非编辑参考只提供可迁移的组织方法，不自动成为证据或模板，也不随全新生成调用上传。先在调用外查看原图，把可迁移信息转写为纯文本提示词，再按以下顺序处理：

1. **核验身份**：记录来源、作者或仓库、链接、版本和许可证。无法核验时只标为风格参考，不引用其科学内容。
2. **双尺度查看**：查看原始分辨率，再按目标载体尺寸查看；只在大图成立的微小细节不迁移。
3. **分层提取**：分别记录核心问题、拓扑、阅读顺序、视觉权重、配色角色、文字密度、微型图元和载体适配。参考图的宽高比、节点宽度和字体形态只作为观察项；目标载体不匹配或存在压窄时记入反例，不迁移。
4. **内容与样式分离**：保留可迁移的结构和视觉逻辑，不复制论文专属数据、公式、标签、机构标识或装饰特征。
5. **反例记录**：同时写明不可复制的拥挤、伪文字、错误标注、无来源机制、固定配色和生成痕迹。

解构记录使用：`来源身份 → 图的任务 → 为什么有效 → 可迁移规则 → 不可复制内容 → 最终尺寸结论`。所谓 Nature、Cell、CVPR 或 NeurIPS 风格不能替代目标期刊正式要求，也不能作为事实依据。

### 5.2 载体定位与实例隔离

Word、PPT、PDF、HTML 或导出包中的媒体文件名只是存储标识，不是图的语义身份。选择 Image 1 前按以下顺序定位：

1. 用户明确指向的图号、图题、页面、幻灯片或正文位置。
2. 图题与正文交叉引用、替代文本、相邻段落和页面顺序。
3. 文档关系、媒体绑定和渲染后主体、标签、比例与配色的视觉核对。
4. 提取文件名或媒体序号只作为回到实际文件的定位线索，不据此推断正式图号。

语义身份与存储名称冲突时，以载体语义身份和渲染核对为准；仍有多个候选时先并排查看，只有会改变编辑目标且无法消歧时才向用户提出一个问题。

通用 skill 只保存定位方法、字段和验收规则。图号、文件名、标签、方法名、阈值、比例、配色和单次修正等项目专属事实只进入本次运行时合同或项目记录，不回写通用 skill、references 或示例。图号、图题、论文标题和来源说明由载体管理时，记录为载体管理文字，不烧录进图片。

### 5.3 携图编辑合同

需要修改既有图片、根据既有图重绘或修正上一版成图时，先建立缩略编辑合同：

| 字段 | 要求 |
| --- | --- |
| Baseline / Image 1 | 唯一编辑目标与验收基线；保留原件，记录来源和当前版本 |
| Image 2 | 可选辅助参考；只允许传递经核验的配色、空间组织、阅读方向和线条处理；不传递宽高比、节点宽度、窄体字体或压缩变形 |
| 载体语义身份 | 用户指向、图题或正文交叉引用、页面位置与文档关系共同确认的真实目标 |
| 项目专属事实 | 本次来源支持的标签、方法、阈值、比例、配色和修正要求；只在本次合同中使用 |
| 载体管理文字 | 由 Word、PPT、网页或排版系统承担、不得烧录进图片的图号、图题、标题或来源说明 |
| 单一修改 | 本轮只解决一个主要问题，写成可观察的变化 |
| LOCKED | 不得改变的文字、数字、公式、对象、人物身份、节点、连线、方向和结论 |
| FLEXIBLE | 允许优化的版式、间距、字体层级、配色、线宽、背景和视觉重心 |
| FORBIDDEN | 不得添加、删除、纠正、推断或合并的内容 |
| 核验基线 | 初始原图、上一轮结果和最终载体尺寸 |

- 所有待附图片都有本地路径时使用 `referenced_image_paths`；近期对话能覆盖全部待附图片时使用最小 `num_last_images_to_include`；两者不得并用。单一机制无法覆盖全部图片时，省略非必要 Image 2；Image 2 对修正确有帮助时要求重新附图。
- Image 1 提供全部内容与身份；用户已提供或指定且参考能改善视觉语言、布局节奏或结构表达时加入 Image 2，禁止传递对象、标签、数据、错误布局或品牌。上一轮结果继续修改时，将其作为当前目标，但仍与 Image 1 核对累积漂移。
- 按 `事实语义 → 精确文字/数字/公式/身份 → 对象/节点/箭头/关系 → 最终尺寸可读性 → 视觉精修` 验收。候选图更美观但任一高优先级变差时，保留 Image 1。
- 最多连续两轮成功返回候选图的质量修正；适用的 Image 2 可从首轮开始使用，尚未使用时必须占用其中至少一轮。可替代的生成稿仍失败时，才按完整内容合同纯文本整图重生一次；不可替代的用户原图不得静默重造。SVG 只能位于这些适用 imagegen 路径之后。
- 统计图回到分析代码修改，科研原始证据只做允许的确定性处理，均不得进入生成式携图编辑。
- 第一次 HTTP 524 用同一目标和保留内容锁的压缩提示词重试一次；第二次 524 停止并保留 Image 1，不把超时计为设计失败，不静默降级模型或切换 SVG/API。

## 6. 结构化提示词包

提示词不设固定字数。写到足以消除结构歧义即可，复杂不等于冗长。生成前整理以下字段：

```text
Figure ID: <候选 ID>
Source anchors: <文件/章节/表图/results.yaml 键/核验文献>
Core claim: <一句核心命题>
Evidence class: <证据属性>
Information role: <正文内容图 / 真实截图 / 氛围插图；其它证据类按原路线>
Carrier and final size: <载体、比例、实际占位>
Geometry and typography: <可用宽度、节点/侧栏容量、必要的重排方式、正常字宽、保持宽高比、禁止非等比缩放>
Template contract: <必须保留 | 允许调整 | 内容驱动的版式变化>
Archetype and reading order: <结构原型、主方向>
Regions: <区域 ID | 比例位置 | 任务 | 内容>
Micro-visuals: <模块 ID | 与真实操作对应的微型图元；无则 none>
Icon strategy: <none / per-major-stage / semantic anchors only；数量、复杂度与主题映射>
Baseline and image roles: <Image 1 内容基线；Image 2 可选且仅提供经核验的视觉与布局辅助>
Target identity and instance facts: <载体语义身份、实际 Image 1、项目专属事实、载体管理文字>
LOCKED / FLEXIBLE / FORBIDDEN: <精确内容锁、视觉自由和禁止变化>
Nodes and edges: <精确节点、边、方向、关系>
Text (verbatim): <必须逐字准确的短标签>
Text selection: <身份标签 | 必要关系/条件 | 少量消歧限定语 | 移到正文/图注/长描述的解释>
Palette roles: <常规路径/辅助或对照/警示或关键状态；默认两类主色相，必要时第三类，主色相总数不超过三种；黑白灰中性色不计>
Safe zones: <标题、正文、按钮、裁切和图注区域>
Must not infer: <不得生成或推断的内容>
Avoid: <本任务的反例与生成痕迹>
```

完整论文生成多张图时，每个候选必须保留 `source anchors`、`core claim`、`rationale` 和 `suggested aspect ratio`，不得只保存一段脱离来源的长提示词。

## 7. 四轮签发

### Pass 0: 必要性与证据

- 候选满足必要性条件；来源锚点真实存在。
- 统计图、原始证据和解释示意路由正确。
- 无虚构结论、数值、公式、解剖、期刊要求或来源。
- 真实 UI、终端、代码、文档和成果示例已路由到实际渲染截图，不由 imagegen 伪造。

### Pass 1: 内容与拓扑

- 核心命题、内容链和区域贡献一致。
- 节点、文字、数字、箭头、方向、分支、合并和反馈逐项准确。
- 多面板无重复问题、重复数据或无贡献填充面板。
- 编辑候选与 Image 1 对照后，关键文字、身份、节点和边达到对应图类的完整保留门槛。
- 正文内容图含理解结构所需的准确短标签；氛围图才把标题与正文全部交给载体。

### Pass 2: 视觉与可读性

- 原始分辨率下无伪文字、重复对象、解剖或物理错误。
- 层级、主次、负空间、配色角色和微型图元有语义。
- 流程图主色相不超过三种，通常使用两类语义色；第三类仅对应真实警示、失败、异常、不良结局或关键状态。
- 最终尺寸下文字、箭头、轮廓和面板仍可辨识；颜色不是唯一线索。
- 画布比例有载体依据；中文字形、圆形和方形保持自然几何，节点未因单行过载而压窄。
- 模板已按内容拓扑调整结构、层级、密度、留白和图文比例，没有机械套版或把内容硬塞进预设卡片。

### Pass 3: 最终载体

- 已嵌入真实 PPT、论文页、标书、报告或网页渲染检查。
- README、技术文档和教程已检查桌面与窄屏显示、真实截图清晰度、替代文本和文件体积。
- 比例、裁切、安全区、图注、替代文本、打印或投影均符合载体。
- 候选图通过所有高优先级检查后才替换 Image 1；美学改进不能补偿事实、精确内容、结构或可读性退化。
- 所有 must-fix 闭环后才能签发；建议项可以记录但不得掩盖失败。
