# 载体视觉规范

目标尺寸和比例服从真实容器。以下为无明确模板时的起点，不替代期刊、基金机构、品牌或项目的正式要求。

## 0. 比例与几何完整性

目标宽高比由最终容器决定，不由生成工具默认值、既有候选图或 Image 2 决定。文字密集的流程、架构、路线和图形摘要在没有明确载体约束时，优先从 4:3 或 3:2 开始；16:9 只在实际图位需要且内容容量足够时使用。宽于 16:9 的比例必须有真实横幅或整行图位依据，并在生成前通过横向容量检查，不能用超宽画布掩盖节点过多或侧栏过重。

生成前做一次几何预算：`左右安全边距 + 节点自然宽度 + 节点间距 + 箭头 + 侧栏 <= 可用宽度`。节点自然宽度按最长一行标签的正常中文字宽和必要内边距估算，不允许预设窄体字、压缩字距或横向缩放。单行包含 6 个及以上有字节点、同时存在解释侧栏，或任一标签只能靠挤窄才能放下时，必须比较双行主链、阶段分组、上下布局、主图加图外说明或拆图；只有自然字宽和清楚箭头均成立时才保留单行。

图片生成、派生和嵌入均锁定宽高比，禁止非等比缩放。容器比例不一致时，选择 `contain` 加留白、语义安全裁切或按目标比例重新生成；不得拖拽宽度或高度单独适配。验收时同时看中文字形是否保持正常方正比例、圆形是否仍为圆形、正方形是否仍为正方形；任一出现横向压窄或纵向拉长即退回重排或重生。

## 1. PPT

PPT 中的科研图先阅读 `research-figure-patterns.md`，再按页面功能和实际占位生成。先做逐页视觉决策，不以“每页有图”作为完成标准。

### 视觉角色

- 封面与主视觉：建立主题识别，不复述标题。
- 背景页：呈现真实问题情境或研究对象。
- 方法页：解释流程、系统、技术路线或关键操作。
- 结论页：突出一个核心发现或影响，不生成虚构数据图。

### 构图与输出

- 整页优先 16:9；左右图文按实际图片区生成 3:2、4:3 或竖向图，不把方图硬塞入宽框。
- 单一概念图通常占侧栏 30%–45%；机制图占 45%–60%；分步方法采用文字 30%–35% 加图 55%–65%；技术路线占标题以下 70%–90%。
- 背景页优先在上方写核心论断，在下半部 35%–55% 放真实证据或解释图；两项证据左右等高排列并分别题注。
- 为标题与要点保留干净负空间，主体、视线和运动方向朝向页面内部。
- 默认不在主视觉内写标题、正文或页码；这些由 PPT 原生文本承担。
- 生成后嵌入真实模板，导出 1280×720 或更高的逐页 PNG，检查缩略图、全屏和投影对比。
- 图片需有替代文本；颜色不能是唯一的信息线索。
- PPT 技术路线的 icon 只作少量语义锚点：常规 5–7 阶段通常使用 2–4 个，复杂多分支使用 0–3 个；小于最终可辨识尺寸时删除，具体按 `diagram-iconography.md`。
- 外部图和证据图保留可追溯来源；签发前检查水印、缺图占位符、低清截图、伪文字和混杂画风。

## 2. 论文与图形摘要

### 可生成内容

- 概念框架、机制示意、科学教育插图、研究流程、图形摘要中的非数据视觉和研究场景。
- 统计图、坐标图、森林图、热图、效应量和任何真实数据映射仍由 `publication-figures` 生成。
- 显微、病理、医学影像、凝胶、临床照片和其它原始证据不得生成式重绘或补全。

### 排版与技术

- 先确定单栏、双栏或整页的最终物理尺寸，再生成或导出；不要先做任意大图后强行缩小。
- 无期刊规则时，位图按最终尺寸至少准备 300 ppi；复杂线条和小字可能需要更高分辨率。目标期刊要求优先。
- 图内文字通常使用清楚的通用字体，最终可读字号按期刊要求；PLOS 的公开示例为 8–12 pt、300–600 dpi，仅作为核对参考，不作为所有期刊通则。
- 图号、标题和长图注放在正文或投稿系统，不烧录进图片；图形摘要只保留完成理解所需的短标签。
- 去除多余空边，但保留防裁切安全边距；不得拉伸、插值伪造清晰度或改变原始证据图像的信息。

### 图形摘要

- 只表达一条主命题：对象或人群 → 方法或机制 → 关键结果 → 含义。
- 统计数字必须来自 `results.yaml` 或经核验来源；不得让 imagegen自行生成或改写。
- 标签、数字与箭头必须逐项核验。Image 1 定向修正、适用的 Image 2 辅助修正与允许的整图重生成均已完成后仍不准确时，才以 `svg-diagrams` 整图最终回退。
- 遵守目标期刊的比例、文件格式、字体、颜色和文字量要求；没有核验不得声称符合期刊规范。

## 3. 基金标书

### 视觉角色

- 立项依据：问题规模、现有证据与关键缺口。
- 科学问题：变量、对象或层级之间的核心关系。
- 创新点：现有路径与拟议方案的本质差别。
- 研究内容与技术路线：任务包、依赖、验证和产出。
- 预期影响：从研究产出到临床、公共卫生或政策价值的路径。

### 构图与验收

- 一张图只服务一个评审问题，优先页面宽度或 3:2、4:3 横图；避免为装饰占用有限版面。
- 先保证 100% 页面缩放下可读，再考虑局部精美细节；标题和解释尽量留在标书正文。
- 关系图使用短标签、明确方向和有限层级；创新点用视觉对比，不用夸张光效或宣传语。
- 配色与标书正文统一，并检查灰度打印和普通办公打印后的区分度。
- 任何样本量、效应、发生率、时间表或结果必须来自可核验来源；示意性内容明确保持为示意，不包装为已有证据。

## 4. 报告

- 封面和章节图采用编辑出版构图，通常用 3:2、4:3 或竖向比例；图片与标题区分工，不把报告标题烧录进图。
- 分析报告中的结果图走 `publication-figures`；概念、机制、场景、流程和章节视觉走本技能。
- 文献、证据和审查主题可用纸张、油墨、档案、索引与批注语言；临床和现场主题采用自然光纪实或准确科学插画。
- 按 Word 版心的真实宽度检查，避免大图缩成不可辨识的小插图。最终 PNG 应保持清楚边缘和合理文件体积。

## 5. 网页

### Hero、横幅与登录侧栏

- 桌面端常用 16:9 或 3:2；移动端优先另生成 4:5、3:4 或适合实际容器的版本，不依赖同一张图极端裁切。
- 主体放在安全区，为标题、正文、按钮和导航留出明确负空间；不得把网页标题或按钮文字烧录进图片。
- 画面需适应 `object-fit: cover` 的裁切逻辑，关键对象不能贴边。

### 卡片、内容图与空状态

- 卡片常用 1:1 或 4:3，一个主对象加最多一个辅助对象，在 320 px 级别仍能辨认。
- 48 至 96 px 的控件图标优先使用已有 Lucide 或项目图标系统；imagegen 用于定制内容插画，不替代保存、返回、搜索等熟悉符号。
- 空状态插画应解释当前状态或下一步，不制造无关装饰场景。

### 技术与可访问性

- 保留高质量源图，再派生 WebP/AVIF 或项目支持的压缩格式；不得把数兆字节原图直接用于小卡片。
- 使用响应式图片资源和明确尺寸，避免布局偏移；桌面与移动端需要不同时采用 art direction。
- 信息图、功能图和内容图提供有意义的替代文本；纯装饰图片使用空替代文本。不要把关键说明只放在图内。
- 在真实网页上检查桌面和移动视口、浅色与深色背景、文字覆盖、裁切、加载体积和清晰度。

## 6. README 与技术文档

### 图位类型

- **项目 hero**：建立主题识别和情境，不复述 README 标题；默认无字，为原生标题、徽章和导航保留空间。
- **正文内容图**：工作流、架构、能力路径、skill 示例和前后对照必须含理解内容所需的短标签和关系，不做无字装饰插图。
- **真实截图**：安装、命令、界面、代码、文档、PPT、表格和交付物预览优先使用实际运行或渲染结果；不得用 imagegen 生成似真截图。
- **成果展示**：真实截图、真实统计图和概念图按证据属性并列，使用共同基线、统一边缘和相近视觉重量，不混成不可追溯拼贴。

### 结构与文字

- README 小节标题已经承担图题时，正文内容图不重复长标题；只把识别节点、阶段、输入、处理、产物和关系所需的最小充分文字与箭头直接入图。
- Skill 示例至少回答“什么输入 → 使用哪个能力 → 形成什么产物”；只画抽象道具而无文字映射，不能证明 skill 能力。
- 教程优先原生代码块加连续真实截图。只有步骤间关系比界面细节更重要时，才改用带准确标签的内容图。
- 同一页面不把所有场景都做成三列卡片。顺序流程、比较、层级、截图序列和成果集合分别选用主链、对称网格、嵌套、编号步骤或共同基线。

### 技术与验收

- 主图宽度通常按 README 内容列生成；并排示例在约 300–420 px 宽时仍应辨识，内容图小字不可依赖点击原图才能读。
- 保留高质量源图，再派生项目支持的 PNG、WebP 或 AVIF；明确尺寸并控制文件体积，避免数兆字节小卡片图。
- 为内容图和截图写能传达目的与结论的替代文本；纯氛围 hero 可使用简短描述，纯装饰图使用空替代文本。
- 在 GitHub 或目标文档渲染器中检查桌面与窄屏：表格折行、图片并排、暗色模式边缘、裁切、加载和清晰度均需实际查看。

## 7. 跨载体派生

同一视觉概念可以派生多种构图，但不能只缩放同一张图：

| 母版 | 派生方向 |
| --- | --- |
| 论文概念图 | PPT 全页版增加负空间；网页版去掉图内长标签；标书版突出逻辑路径 |
| PPT 主视觉 | 报告封面改为竖向或 3:2；网页 hero 另做移动端构图 |
| 标书技术路线 | 论文版压缩标签并按期刊尺寸；PPT 版扩大字号和阶段间距 |
| 网页卡片图 | 不直接放大成 hero；重新组织主体、支持元素和背景层次 |
| README 内容图 | PPT 版扩大字号并减少节点；网页卡片版只保留一个功能；技术文档版可增加真实截图与步骤标注 |

共享主题、主色来源、材质和视觉语言，分别重算比例、主体尺度、负空间和信息密度。
