# Imagegen 科研视觉工作流

把 imagegen 作为科研非统计视觉资产的默认生成路径，而不是固定画风生成器。先判断载体、读者、信息功能、证据属性、信息角色和实际显示尺寸，再选择主体、文字策略、视觉语言、材质、构图、配色与细节密度。科研感必须来自真实主题和关系，不来自 DNA、显微镜、试管、冷蓝界面或公式堆砌。模板是可调整的约束集合，不是不可改变的构图答案；科学性、内容清晰度和最终可读性优先于套版一致。

Codex 使用内置 `imagegen` 技能及其 `image_gen` 工具。不要另写 API 脚本，不猜模型参数，不把 CLI/API 当作内置工具失败后的静默替代。其它平台只使用当前会话明确提供且已配置的图像生成工具。

## 必读资源

- 每次生成前读 `references/visual-strategy.md`，确定视觉角色、风格、构图和跨资产一致性。
- 按实际载体读 `references/carrier-specs.md` 的 PPT、论文、标书、报告或网页章节。
- README、技术文档、项目主页、教程、skill 示例、真实截图、成果展示或已有模板任务，完整阅读 `references/scenario-playbook.md`，先选内容图、截图或氛围图路线，再确定文字与模板策略。
- 输入是完整论文、项目材料、多图系列、复杂多面板或参考图时，先读 `references/figure-planning.md`，完成来源到图件矩阵、图前合同、参考图解构和四轮签发。
- 生成流程图、框架图、机制图、研究设计、技术路线或图形摘要前，完整阅读 `references/research-figure-patterns.md`，锁定图类的内容契约、证据属性和视觉语法。
- 流程图、研究设计、技术路线、方法架构或图形摘要需要决定是否使用 icon 时，读取 `references/diagram-iconography.md`；先判定不用图标是否更清楚，再确定数量、复杂度、主题映射、统一风格和配色。
- 内容契约锁定后读 `references/prompt-recipes.md`，选择对应提示词配方。
- 计算模型、网络架构、复杂多面板或需要扩展提示词词汇时，先读 `references/external/SOURCE.md` 的调用规则，再按需检索所列上游原文；不得把上游文件当独立 skill 直接执行。
- 同时遵循内置 `imagegen` 的生成、编辑、保存路径、目标图选择和透明背景规则；全新创作走纯文本生成，修改既有图或修正上一版走携图编辑。

## 证据属性检查

先给目标视觉归类，不能为了“更美观”跨越证据边界：

1. **证据型原始图像**：显微、病理、影像、临床照片、仪器输出、原始截图、凝胶或其它研究证据，不得由 imagegen 新造、补全、净化或重绘。仅做期刊允许且不改变信息的确定性处理，并保留原始文件。
2. **统计数据图**：任何坐标、比例、效应值、置信区间、样本量或真实数据映射，调用 `publication-figures`，不得让 imagegen 虚构。
3. **精确结构图**：流程、机制、框架、路线、图形摘要先用 imagegen 生成包含最小充分文字及必要数字、箭头的整张图；逐字逐边核验，适用的 Image 2 辅助修正与其它允许的 imagegen 路径均已实际尝试后仍不合格，才整图回退 `svg-diagrams`。
4. **解释与编辑插图**：科学教育插图、研究场景、概念配图、封面、网页主视觉和标书配图默认用 imagegen。
5. **界面控制图标**：按钮、工具栏、导航和状态控件已有 Lucide 或项目图标系统时直接复用，不生成位图替代；定制功能插画、内容卡片和空状态可用 imagegen。
6. **真实界面与成果截图**：软件界面、终端、代码、文档、PPT、表格、实际图件和交付包示例优先运行或渲染真实产物后截图；不得让 imagegen 生成看似真实但不可复核的 UI、命令、论文页或结果预览。

## 不降质合同

图片编辑和重绘必须把 Image 1 视为验收基线，而不是可覆盖的临时输入。提示词只能降低风险，不能保证候选图更好；原图始终保留，候选图只有通过全部验收条件后才成为当前版。

按以下优先级比较 Image 1 与候选图，前项不能用后项补偿：

1. 事实与语义忠实度。
2. 文字、数字、公式和人物或对象身份的精确性。
3. 对象、节点、箭头、分支、汇合和关系的正确性。
4. 最终显示尺寸下的可读性。
5. 视觉精修。

候选图更美观但任一高优先级项目变差时，判定失败并保留 Image 1。不得覆盖、删除或改名原图，直到候选图通过对应图类的全部 acceptance checks。

参考图角色必须显式锁定：

- **Image 1**：唯一编辑目标与验收基线，科学内容、身份和结构从此图继承。
- **Image 2**：可选辅助参考。用户已提供或指定且它能改善视觉语言、布局节奏或结构表达时使用，只传递经核验的配色、空间组织、阅读方向和线条处理；不得传递对象、标签、数据、错误布局或品牌，也不得让它决定目标画布比例、节点宽度、字体宽度或任何压缩变形。
- 不为凑足图片数量寻找或生成 Image 2；但适用的 Image 2 已存在时，必须先于 SVG 用于至少一轮携图修正。不得让 Image 2 覆盖 Image 1 的内容合同。

## 开始前检查

- 命中本技能 `description` 后，必须先加载本技能与所需 references，再选择实现路径。当前可见 skills 清单未列出本技能，不等于 `image_gen` 不可用，也不是转 SVG 的依据。
- PPT 任务先为每页指定主要表达方式。触发本技能不等于每页都要生成图片；目录、章节、目的、定义、公式、创新、可行性和计划页若用模板、文字、表格、公式或时间轴表达更清楚，应明确不生成装饰插图。
- README、技术文档和网页任务先为每个图位标记 `内容图/真实截图/氛围图/统计图/原始证据/无需图片`。正文中的工作流、架构、技能示例和成果路径不得误判为无字装饰插图。
- 信息足够时直接执行。只有缺失项会明显改变内容、比例或留白方向时，最多提出一个必要问题；其余按载体和现有视觉系统作保守判断。
- 通用 skill 只保存可迁移的方法、字段和验收规则。项目专属的正式图号、导出文件名、标签、方法名、阈值、比例、配色和单次修正只进入本次运行时合同或项目记录，不写回本技能或 references。
- 用户要求实际生成图片、PPT、论文、标书、报告或网页时，不得停在方案、提示词或“正在运行”。必须完成生成、查看原图、修正、保存、嵌入和最终载体检查。

## 核心工作流

0. **规划图件集合**：完整论文、项目或多图请求先按 `figure-planning.md` 建立来源到图件矩阵。只保留来源明确、具有独立信息贡献且能在最终尺寸成立的候选；不按章节数量机械配图。单张明确请求执行缩略图前合同即可。
1. **读取载体环境**：检查模板、页面或网页的比例、占位、背景、配色、字体、相邻文字、按钮位置和已有图片。载体内嵌图先按用户指向、图题与正文交叉引用、页面或段落位置、文档关系和渲染外观定位；导出文件名和媒体序号仅作存储线索，不得从中推断正式图号。PPT 先按页型标记 `插图/统计图/原始证据/表格/公式/流程图/时间轴/纯文字`；README、技术文档和网页按图位标记 `内容图/真实截图/氛围图/统计图/原始证据/无需图片`；没有现成视觉系统时才建立新的系统。
2. **锁定信息功能与角色**：只选一个主要功能：解释关系、建立情境、提示功能、区分章节、突出创新、展示路径或形成品牌识别；再选一个信息角色：`正文内容图/真实截图/氛围插图`。内容图必须自含理解所需的短标签和关系；截图必须来自真实可复核产物；氛围图才把标题和正文留给载体原生排版。
3. **审查模板适配与横向容量**：把模板拆成 `必须保留` 与 `允许调整`。保留尺寸、品牌、字体、核心色和必要组件；按 `carrier-specs.md` 先用实际图位核定宽高比，再根据最长标签的自然字宽、节点内边距、箭头、间距和侧栏估算横向容量。模板迫使流程变成卡片墙、比较失去对称或层级变平，或 Image 2 导致节点过窄和文字压缩时，改为双行、分组、上下布局或拆分说明，不得靠超宽画布、窄体字或非等比缩放硬塞。
4. **形成视觉简报**：在内部写不超过 120 字的规格，包含载体、读者、一句核心命题、来源锚点、信息角色、文字策略、主体、风格、构图、模板适配、配色来源、材质、目标比例、几何完整性、负空间和禁止项。不得先套固定画风。
5. **锁定内容契约与文字预算**：从来源锚点核验对象名称、关系语义、数字和结论，再列出各区域的独立贡献，以及必须出现、必须逐字准确、可以抽象、不得出现和不得推断的对象、文字、数字与关系。来源没有的结论、期刊标识、仪器读数或医学细节不得补写；移除后不影响核心命题的区域直接删除。正文内容图只把删除后会造成对象、关系、条件、关键状态或核心结论误读的文字列入 `Text (verbatim)`；定义、理由、完整方法、限制和来源移到正文、图注或复杂图长描述。不得用“无伪文字”错误地删除必要标签，也不得给每个节点机械添加副标题。
6. **建立视觉系统**：同一项目的系列图共享主色来源、光线方向、材质、线条或摄影语言；不同内容拓扑和载体按实际尺寸重算结构、密度和裁切，不能把同一模板机械复制到所有图片。流程图默认使用两类有明确含义的主色相，只有真实警示、失败、异常、不良结局或必须单独识别的关键状态才增加第三类；全图主色相不超过三种，黑、白、灰等中性色不计入。不得把蓝白配色作为科研流程的默认答案，也不得为了避免单调而随机换色。流程与路线图同时锁定 icon 策略：默认无图标，只有真实对象、操作或产出需要视觉锚点时才加入少量简单 icon 或微型图元。
7. **构造提示词**：使用 `figure-planning.md` 的结构化提示词包和 `prompt-recipes.md` 的对应配方，至少保留来源锚点、核心命题、证据属性、信息角色、模板适配、结构原型、区域、语义微型图元、icon 策略、精确节点与边、逐字标签、配色角色、目标宽高比、自然字宽与禁止非等比缩放、安全区和禁止推断项。用具体视觉语法替代“高级、科技感、学术感”等空泛词，不设机械字数下限。
8. **生成或截图首稿**：概念性非统计视觉首轮只生成一个完整方案；真实界面与成果先运行、渲染或打开实际文件后截图，不调用 imagegen 伪造。全新生成不携带目标图；非编辑参考先用于提取拓扑、构图、视觉权重和配色角色并写进完整提示词，候选图成为 Image 1 后，适用参考可作为 Image 2 进入定向修正。项目产物必须移入工作区，不得只留在 `$CODEX_HOME/generated_images/`。
9. **双尺度检查与几何检查**：先按原始分辨率检查伪文字、解剖、纹理、阴影、重复对象、中文字形宽度和圆形/方形是否变形，再按最终显示尺寸检查主旨、轮廓、层级、负空间、对比、裁切、标签和细节密度。嵌入载体时锁定原始宽高比；若目标槽位不匹配，使用留白、裁切或重新生成，不得横向或纵向拉伸。真实截图同时检查敏感信息、光标、通知、无关窗口、缩放和裁切。
10. **携图定向编辑**：需要修改既有图、根据既有图重绘或修正上一版时，先查看 Image 1 并写三段式合同：`LOCKED` 列出不得改变的文字、数字、公式、身份、对象、节点、边、方向和结论；`FLEXIBLE` 列出允许优化的版式、间距、字体层级、配色、线宽、背景和视觉重心；`FORBIDDEN` 列出不得添加、删除、纠正、推断或合并的内容。所有待附图片均有本地路径时使用 `referenced_image_paths`；近期对话能覆盖全部待附图片时使用最小 `num_last_images_to_include`；两者不得并用。单一机制无法覆盖 Image 1 与必要 Image 2 时，省略非必要 Image 2；Image 2 对修正确有帮助时要求重新附图。每轮只修一个主要问题并重复三段式合同；成功返回的候选图必须与 Image 1 全幅对照并通过不降质合同。最多连续两轮候选图质量修正；适用的 Image 2 可从首轮开始使用，尚未使用时必须占用其中至少一轮。仍失败时，仅对可替代的生成稿执行一次纯文本整图重生成。只有这些适用的 imagegen 路径全部完成后精确结构图仍不准确，才整图回退 `svg-diagrams`；不可替代的用户原图保留原件，不以重生图冒充修改成功。
11. **嵌入与载体复核**：把最终图放进实际 PPT、论文页面、标书页、README、技术文档或网页；渲染/截图检查整体，而不是只查看孤立图片。网页和 README 检查桌面与窄屏显示、替代文本和文件体积，PPT 检查缩略图和投影尺寸。
12. **记录与归档**：正式项目记录最终视觉简报、提示词、工具路径、核验结论和回退原因；当前版使用稳定文件名，旧版整组归档。提示词和生成过程不进入正文。

签发按 `figure-planning.md` 依次通过四轮：必要性与证据、内容与拓扑、视觉与可读性、最终载体。所有 must-fix 闭环后才能交付；参考图或外部示例不能替代来源核验。

## HTTP 524

把 HTTP 524 视为携图生成的服务或传输失败，不是候选图质量失败，也不计入两轮设计修正：

1. 第一次 524：使用相同 Image 1、相同必要附件和同一修改目标重试一次；压缩重复形容词和通用禁止项，但保留图片角色、结构 inventory、`LOCKED / FLEXIBLE / FORBIDDEN` 与 acceptance checks。
2. 第二次 524：停止重试，保留 Image 1，不解释为提示词或设计失败；不静默降级模型，不另写或切换第三方 API，也不因超时回退 SVG。

只有 imagegen 成功返回候选图且候选图不满足内容精度要求，才进入后续重生或 SVG 回退判断。

## 美学要求

- 主体必须一眼可辨。小图保留一个主对象，大图通常保留一个焦点和 2 至 4 个支持元素。
- 优先自然不对称构图、稳定视觉重心和有用途的负空间；避免平均分布、机械居中、边缘拥挤和无意义漂浮。
- 配色优先取自现有载体。流程图默认使用两类语义主色，必要时增加一类警示或关键状态色，主色相总数不超过三种；中性色用于画布、文字、边框和连接线。红色只用于真实警示、失败、异常、不良结局或当前重点，普通排除、常规步骤和一般分组不自动标红。不把整个项目做成单一蓝色相，也不为“高级感”加入霓虹渐变。
- 材质必须有语义：文献与证据可用纸张、油墨和索引；临床与现场可用自然光纪实；机制与系统可用干净科学插画；数字研究可用克制几何和网格。不得无理由加入玻璃、金属、烟雾、景深或电影灯光。
- 不默认使用塑料 3D 图标、黏土风、儿童卡通、商业素材照片感、悬浮全息屏、蓝色发光 DNA、无依据分子结构或科研符号拼贴。
- 系列资产要有家族一致性，但每张图仍服从其内容角色，不能变成同构卡片墙。

## 文字与关系

- **正文内容图**：README 正文图、工作流、架构图、技能示例、教程内容图、流程图、机制图、技术路线和图形摘要必须直接包含脱离图注仍能理解结构所需的短标签、数字和箭头，并由 imagegen 生成完整图后逐字逐边核验。不得把“网页图默认无字”泛化到正文内容图。
- **氛围插图**：PPT 主视觉、封面、网页 hero、登录页和章节氛围图默认不烧录标题、正文、按钮或长解释；这些文字由载体原生排版承担。
- **真实截图**：界面、终端、代码、文档、PPT、表格、图件和交付物预览使用实际渲染内容。只保留任务相关区域，隐藏敏感信息；不得用 imagegen 生成可被误认成真实操作或真实成果的截图。
- 内容图文字、数字和箭头必须作为完整图的一部分由 imagegen 直接生成；不得用 Python、PPT 文本框、Word 文本框或 SVG 覆盖层修补错误文字、数字和箭头。真实截图的确定性裁切、缩放和不改变内容的标注不属于补写 AI 图，但标注不得遮盖证据或改写界面状态。
- 图内标签逐字沿用来源材料、研究方案或可核验方法文献中的规范术语。缩短标签只能删除不影响含义的修饰语，并保留术语核心；不得加入内部流程词、软件工程隐喻、口语化动作、生硬直译、自造缩略语或临时拼接的短语。来源未给出且存在多种合理名称时先向用户核实。
- 图题、图号、论文标题、报告标题和来源说明通常留在载体中，不烧录进图片，除非目标格式明确要求。
- 生成似真论文页面、仪器界面或档案时，只使用不可辨识的短线、点阵和抽象标记，不生成看似真实但错误的段落与数值。
- 修改既有图片、根据既有图重绘和修正上一版成图都属于携图编辑。只传全部且仅必要的编辑目标；上一轮生成结果需要继续修改时，以该结果作为新的编辑目标，同时用初始原图核对漂移。
- 非编辑参考不随全新生成调用上传，先把可迁移规则写入提示词；候选图成为 Image 1 后，若现有参考能改善视觉语言、布局节奏或结构表达，可增加一张 Image 2，但必须锁定其只提供经核验的辅助规则，不提供内容。用户明确要求同时修改多张目标图时才传入多张编辑目标。

## 路由与回退

- imagegen 可用时，流程图、框架图、示意图、图形摘要和跨载体配图不得因“SVG 更容易”而提前跳过。
- 用户明确要求 SVG、矢量源、完全可编辑图形或目标格式强制矢量时，调用 `svg-diagrams`。
- 已检查实际工具后确认不可用，或成功返回的候选图经 Image 1 定向编辑、适用的 Image 2 辅助修正与允许的纯文本整图重生成后仍不能满足精度要求时，才把 `svg-diagrams` 作为最终回退。已有适用 Image 2 却未尝试时，不得转 SVG；HTTP 524 本身也不是 SVG 回退凭证。
- 回退时沿用已锁定的内容契约和视觉简报，不在切换工具时改变信息结构。
