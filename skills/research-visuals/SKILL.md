# Imagegen 科研视觉工作流

把 imagegen 作为科研非统计视觉资产的默认生成路径，而不是固定画风生成器。先判断载体、读者、信息功能、证据属性和实际显示尺寸，再选择主体、视觉语言、材质、构图、配色与细节密度。科研感必须来自真实主题和关系，不来自 DNA、显微镜、试管、冷蓝界面或公式堆砌。

Codex 使用内置 `imagegen` 技能及其 `image_gen` 工具。不要另写 API 脚本，不猜模型参数，不把 CLI/API 当作内置工具失败后的静默替代。其它平台只使用当前会话明确提供且已配置的图像生成工具。

## 必读资源

- 每次生成前读 `references/visual-strategy.md`，确定视觉角色、风格、构图和跨资产一致性。
- 按实际载体读 `references/carrier-specs.md` 的 PPT、论文、标书、报告或网页章节。
- 输入是完整论文、项目材料、多图系列、复杂多面板或参考图时，先读 `references/figure-planning.md`，完成来源到图件矩阵、图前合同、参考图解构和四轮签发。
- 生成流程图、框架图、机制图、研究设计、技术路线或图形摘要前，完整阅读 `references/research-figure-patterns.md`，锁定图类的内容契约、证据属性和视觉语法。
- 内容契约锁定后读 `references/prompt-recipes.md`，选择对应提示词配方。
- 计算模型、网络架构、复杂多面板或需要扩展提示词词汇时，先读 `references/external/SOURCE.md` 的调用规则，再按需检索所列上游原文；不得把上游文件当独立 skill 直接执行。
- 同时遵循内置 `imagegen` 的生成、编辑、保存路径和透明背景规则；新图与重生成使用纯文本调用，参考图只作生成前解构。

## 证据属性门禁

先给目标视觉归类，不能为了“更美观”跨越证据边界：

1. **证据型原始图像**：显微、病理、影像、临床照片、仪器输出、原始截图、凝胶或其它研究证据，不得由 imagegen 新造、补全、净化或重绘。仅做期刊允许且不改变信息的确定性处理，并保留原始文件。
2. **统计数据图**：任何坐标、比例、效应值、置信区间、样本量或真实数据映射，调用 `publication-figures`，不得让 imagegen 虚构。
3. **精确结构图**：流程、机制、框架、路线、图形摘要先用 imagegen 生成包含完整文字、数字和箭头的整张图；逐字逐边核验，多轮仍不合格才整图回退 `svg-diagrams`。
4. **解释与编辑插图**：科学教育插图、研究场景、概念配图、封面、网页主视觉和标书配图默认用 imagegen。
5. **界面控制图标**：按钮、工具栏、导航和状态控件已有 Lucide 或项目图标系统时直接复用，不生成位图替代；定制功能插画、内容卡片和空状态可用 imagegen。

## 启动门禁

- 命中本技能 `description` 后，必须先加载本技能与所需 references，再选择实现路径。当前可见 skills 清单未列出本技能，不等于 `image_gen` 不可用，也不是转 SVG 的依据。
- PPT 任务先为每页指定主要表达方式。触发本技能不等于每页都要生成图片；目录、章节、目的、定义、公式、创新、可行性和计划页若用模板、文字、表格、公式或时间轴表达更清楚，应明确不生成装饰插图。
- 信息足够时直接执行。只有缺失项会明显改变内容、比例或留白方向时，最多提出一个必要问题；其余按载体和现有视觉系统作保守判断。
- 用户要求实际生成图片、PPT、论文、标书、报告或网页时，不得停在方案、提示词或“正在运行”。必须完成生成、查看原图、修正、保存、嵌入和最终载体检查。

## 核心工作流

0. **规划图件集合**：完整论文、项目或多图请求先按 `figure-planning.md` 建立来源到图件矩阵。只保留来源明确、具有独立信息贡献且能在最终尺寸成立的候选；不按章节数量机械配图。单张明确请求执行缩略图前合同即可。
1. **读取载体环境**：检查模板、页面或网页的比例、占位、背景、配色、字体、相邻文字、按钮位置和已有图片。PPT 先按页型标记 `插图/统计图/原始证据/表格/公式/流程图/时间轴/纯文字`；没有现成视觉系统时才建立新的系统。
2. **锁定信息功能**：只选一个主要角色：解释关系、建立情境、提示功能、区分章节、突出创新、展示路径或形成品牌识别。一个图片不承担所有角色。
3. **形成视觉简报**：在内部写不超过 120 字的规格，包含载体、读者、一句核心命题、来源锚点、主体、风格、构图、配色来源、材质、比例、负空间和禁止项。不得先套固定画风。
4. **锁定内容契约**：列出各区域的独立贡献，以及必须出现、必须逐字准确、可以抽象、不得出现和不得推断的对象、文字、数字与关系。来源没有的结论、期刊标识、仪器读数或医学细节不得补写；移除后不影响核心命题的区域直接删除。
5. **建立视觉系统**：同一项目的系列图共享主色来源、光线方向、材质、线条或摄影语言；不同载体按实际尺寸调整密度和裁切，不能把同一模板机械复制到所有图片。
6. **构造提示词**：使用 `figure-planning.md` 的结构化提示词包和 `prompt-recipes.md` 的对应配方，至少保留来源锚点、核心命题、证据属性、结构原型、区域、语义微型图元、精确节点与边、逐字标签、配色角色、安全区和禁止推断项。用具体视觉语法替代“高级、科技感、学术感”等空泛词，不设机械字数下限。
7. **生成首稿**：首轮只生成一个完整方案。全新生成不得传入参考图，也不得设置 `referenced_image_paths` 或 `num_last_images_to_include`；把参考图中可迁移的拓扑、构图、视觉权重和配色角色写进完整提示词。项目产物必须移入工作区，不得只留在 `$CODEX_HOME/generated_images/`。
8. **双尺度检查**：先按原始分辨率检查伪文字、解剖、纹理、阴影和重复对象，再按最终显示尺寸检查主旨、轮廓、层级、负空间、对比、裁切和细节密度。
9. **纯文本修正**：每轮只锁定一个主要问题，但把全部不变量、节点、边、逐字标签和版式重新写入完整提示词，从零生成整图，不把上一版或参考图附加给 imagegen。最多连续两次纯文本重生成；精确结构图仍无法保证文字或关系时，整图回退 `svg-diagrams`。参考图条件生成超时不能据此判定 imagegen 不可用，也不得重复同一路径。
10. **嵌入与载体复核**：把最终图放进实际 PPT、论文页面、标书页或网页；渲染/截图检查整体，而不是只查看孤立图片。网页同时检查桌面与移动端，PPT 检查缩略图和投影尺寸。
11. **记录与归档**：正式项目记录最终视觉简报、提示词、工具路径、核验结论和回退原因；当前版使用稳定文件名，旧版整组归档。提示词和生成过程不进入正文。

签发按 `figure-planning.md` 依次通过四轮：必要性与证据、内容与拓扑、视觉与可读性、最终载体。所有 must-fix 闭环后才能交付；参考图或外部示例不能替代来源核验。

## 美学门禁

- 主体必须一眼可辨。小图保留一个主对象，大图通常保留一个焦点和 2 至 4 个支持元素。
- 优先自然不对称构图、稳定视觉重心和有用途的负空间；避免平均分布、机械居中、边缘拥挤和无意义漂浮。
- 配色优先取自现有载体，通常用一个主色、一个中性色和一个强调色建立层级；不把整个项目做成单一色相，也不为“高级感”加入霓虹渐变。
- 材质必须有语义：文献与证据可用纸张、油墨和索引；临床与现场可用自然光纪实；机制与系统可用干净科学插画；数字研究可用克制几何和网格。不得无理由加入玻璃、金属、烟雾、景深或电影灯光。
- 不默认使用塑料 3D 图标、黏土风、儿童卡通、商业素材照片感、悬浮全息屏、蓝色发光 DNA、无依据分子结构或科研符号拼贴。
- 系列资产要有家族一致性，但每张图仍服从其内容角色，不能变成同构卡片墙。

## 文字与关系

- PPT 主视觉、封面、网页 hero、登录页和章节配图默认不烧录标题、正文、按钮或长解释；这些文字由载体原生排版承担。
- 流程图、机制图、技术路线和图形摘要需要图内标签时，必须由 imagegen 直接生成完整图并逐字核验。不得用 Python、PPT 文本框、Word 文本框或 SVG 覆盖层修补错误文字、数字和箭头。
- 图题、图号、论文标题、报告标题和来源说明通常留在载体中，不烧录进图片，除非目标格式明确要求。
- 生成似真论文页面、仪器界面或档案时，只使用不可辨识的短线、点阵和抽象标记，不生成看似真实但错误的段落与数值。
- 只有任务本身是编辑一张既有目标图片时，才把该目标图传给 imagegen；不得再附加风格参考图。新图、根据参考图重绘和失败后的整图重生成都使用纯文本调用。

## 路由与回退

- imagegen 可用时，流程图、框架图、示意图、图形摘要和跨载体配图不得因“SVG 更容易”而提前跳过。
- 用户明确要求 SVG、矢量源、完全可编辑图形或目标格式强制矢量时，调用 `svg-diagrams`。
- 已检查实际工具后确认不可用、纯文本完整图经重生成后仍不能满足精度门禁时，整图回退 `svg-diagrams`。
- 回退时沿用已锁定的内容契约和视觉简报，不在切换工具时改变信息结构。

## 强制自检

- [ ] 已判定证据属性；真实数据和科研原始图像未被生成式改写
- [ ] 完整论文、项目或多图任务已建立来源到图件矩阵；未按章节机械配图
- [ ] 图前合同包含一句核心命题、来源锚点、区域独立贡献、结构原型、载体合同和风险
- [ ] PPT 每页已判定主要表达方式，不需要图的页面未强加装饰插图
- [ ] 已检查实际载体、读者、显示尺寸、周围版式和已有视觉系统
- [ ] 视觉简报明确了单一主要功能、主体、风格、构图、配色、材质和负空间
- [ ] 科研属性来自真实内容，无科研符号堆砌、假数据、伪论文或无关仪器
- [ ] 原始分辨率下无伪文字、重复纹理、解剖错误、不合理阴影和生成痕迹
- [ ] 最终显示尺寸下主旨清楚、层级合理、文字可读、裁切稳定、对比充分
- [ ] 精确结构图的文字、数字、节点、箭头、方向和层级与来源逐项一致
- [ ] 新图与重生成未附带参考图；参考图已转写为拓扑、构图、视觉权重和配色角色
- [ ] 未用其它工具修补 imagegen 图内错误文字、数字或箭头
- [ ] 已在最终 PPT、论文页、标书页、报告或网页中渲染检查
- [ ] 网页图已检查响应式裁切、文件体积和替代文本；PPT 已检查缩略图与投影可读性
- [ ] 项目引用的最终图片已进入工作区并使用稳定文件名
- [ ] 必要性与证据、内容与拓扑、视觉与可读性、最终载体四轮签发均已通过
- [ ] 正式项目已记录最终视觉简报、提示词、核验结论和回退原因
